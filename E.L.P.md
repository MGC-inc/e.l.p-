## 3. システム構成（ホワイトボード設計の写し）

```
 LINE ──┐
 Slack ─┤
        ▼
  Claude Code ◄────► GitHub（このリポジトリ＝単一情報源）
                        ▲  ▲  ▲
                google ─┘  │  └─ タスク管理
                       イノベラ
```

- **GitHub（このリポジトリ）**: E.L.P事業内容・KPI・従業員情報・MTG議事録などの単一情報源
- **Claude Code**: 自然言語の指示でこのリポジトリを読み書きする
  - 例:「KPIをもとに〇〇さんのHowを詰めたい」→ `KPI.md` + `従業員.md` を参照
  - 例:「〇〇とのMTG議事録を踏まえて△△して」→ `議事録/` を参照
- **イノベラ**: 電話録音 → 要約 → LINEで担当者へ通知（フローは [イノベラ.md](./イノベラ.md)）

## 4. 運用ルール

1. **情報はこのリポジトリに集約する** — チャットやローカルメモに散らさない
2. **1ファイル1テーマ** — 肥大化したら分割する（例: `KPI.md` → `KPI/2026.md`）
3. **更新は都度コミット** — `docs(KPI): 6月目標を更新` のように Conventional Commits 形式
4. **APIキー・パスワード等のシークレットは絶対にコミットしない** — 保管場所の参照のみ `API一覧.md` に記載
5. **議事録は命名規則に従う** — `議事録/YYYY/YYYY-MM-DD_相手_テーマ.md`
