# CLAUDE.md — E.L.P 組織管理リポジトリ

このリポジトリは株式会社E.L.P（株式会社クリーンマン）の組織情報の単一情報源。コードではなくMarkdownドキュメントの管理が目的。

## 必ず最初に読む

[E.L.P.md](./E.L.P.md) — 全ドキュメントのマップと運用ルールが書いてある。

## リクエスト → 参照ファイルのルーティング

| ユーザーの依頼例 | 参照・更新するファイル |
|---|---|
| 「KPIをもとに〇〇さんのHowを詰めたい」 | `KPI.md` + `従業員.md` |
| 「〇〇とのMTG議事録を踏まえて△△して」 | `議事録/YYYY/` 内の該当ファイル |
| 「議事録を作成して」 | `議事録/_template.md` をコピーし `議事録/YYYY/YYYY-MM-DD_相手_テーマ.md` で作成 |
| 「イノベラの通話を要約して通知」 | `イノベラ.md` のテンプレ・送信先に従う |
| 「〇〇のAPIキーはどこ？」 | `API一覧.md`（保管場所の参照のみ。キー値は扱わない） |
| 「担当者は誰？」「どんな人？」 | `従業員.md` |
| 「〇〇社との経緯を踏まえて」 | `顧客一覧.md` → `議事録/`・`通話ログ/` を顧客名で検索 |
| 「先週の電話の内容は？」 | `通話ログ/YYYY/` 内の該当ファイル |
| 「未完了タスクを一覧にして」 | `タスク.md` |
| 「〇〇のやり方・手順は？」 | `マニュアル/` 内の該当ファイル |
| 「マニュアルを作って」 | `マニュアル/_template.md` をコピーし `マニュアル/業務名.md` で作成 |

## ルール

1. **シークレット禁止**: APIキー・トークン・パスワードの値をこのリポジトリに書かない。`API一覧.md` には保管場所の参照のみ
2. **機微情報禁止**: 従業員の住所・電話番号・給与は書かない
3. **日付形式**: `YYYY-MM-DD`。議事録ファイル名は `YYYY-MM-DD_相手_テーマ.md`
4. **コミット形式**: Conventional Commits（例: `docs(議事録): 2026-06-10 東さんMTG追加`）
5. **1ファイル1テーマ**: 肥大化したらディレクトリに分割し、`E.L.P.md` のドキュメントマップを更新する
6. **新ファイル追加時**: 必ず `E.L.P.md` のドキュメントマップに行を追加する
