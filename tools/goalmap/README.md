# tools/goalmap — ゴールマップ図解レンダラー

各メンバー1人分の目標データ（JSON）から、「下から上へ登る地図」の図解（SVG / PNG）を生成する。
週次MTGの運用ルールは [`../../ゴールマップ運用.md`](../../ゴールマップ運用.md) を参照。

三つの使い方:
- **`goalmap-studio.html`（MT中の主役）** — ブラウザで開くだけ。左でチェック／編集 → 右の図解と達成率が即更新。MTが終われば翌週の図解が完成。保存はブラウザ＋JSON書き出し、PNG/印刷で出力。
  - **メンバー切替**：ヘッダーの**ドロップダウン**、または直下の**アバター・ロスター**（全員の今のアバター＋名前＋達成率を横並び。クリックで切替、選択中はハイライト）。岡野・宮腰は**ファイル内にデータ内蔵**済みなので、ダウンロードしてダブルクリックすれば最初から並ぶ（他メンバーは「JSON読込」or「＋新規」で追加）。
  - **↺ 初期化**：表示中メンバーを内蔵の初期値に戻す（編集を間違えた／古いデータが残った時のリセット）。
  - **アバターPNG**：いまのアバター（成長キャラ）だけを1枚のカード画像で書き出す。朝礼・Slack・Notionに「今週の○○さん」として貼る用。
  - 各人の最新編集はブラウザ(localStorage)に自動保存。**MT終了時に「JSON保存」**して `members/<名前>.json` を更新するのが“正”の記録。
  - **スマホ対応**：画面幅760px以下では自動で1カラムに切り替わり、**地図（図解）を上・編集フィールドを下**に積む（横スクロールなし／タップ前提でチェックや入力を拡大、iOSのフォーカス時ズームも抑止）。出先・スマホで「見る／チェックする」運用も可能。オフライン（ファイル単体）でも同じ。
  - **📱 1枚表示（週次スクショ共有用）**：ヘッダーの「📱 1枚表示」で、**ゴール・小さいアバター・達成率・現在地・今週の最優先タスク・全タスクのチェックを1画面に凝縮**したスマホ縦サイズ（430px幅）のカードに切替。ロスターは自動で隠れる。**この状態で「PNG出力」を押すと、その1枚カードだけが画像で書き出せる**（週次の共有スクショに最適）。もう一度押すと通常の地図に戻る。
  - **LINE向け画質**：PNG出力は高解像度（scale 3）。LINEは「写真」送信で圧縮されるので、**文字までクッキリ見せたい時は「ファイル」送信**を推奨（原画質キープ）。1枚カードはLINE可読性に合わせて文字を一回り大きくしてある。
  - **週次MTでの編集**：①できたタスクにチェック ②遅れたらフェーズの期限を延期（日付変更／「＋1週」で**そのフェーズ以降をまとめて1週うしろへ**） ③タスク内容はその場で書き換え・追加・×削除 ④ステージ変更。**期限超過は赤＋⚠**で強調。
- **`generate_goalmap.py`（バッチ／記録）** — JSON → SVG/PNG をコマンドで一括生成。`--summary`（または `-s`）で**スマホ1枚カード**（`out/<名前>_1枚.png`）を出力。
- **`summary_cards.py`（週次1枚カードを全員分）** — `members/*.json` 全員の1枚カード（ゴール・小アバター・現在地・今週の最優先・全タスク✓）を `out/summary/<名前>.png` に出力。LINE等で一括共有用。studioの「📱 1枚表示」と同仕様。
- **`team_chart.py`（チーム達成率の棒グラフ）** — 全員の達成率を横棒グラフ1枚（`out/チーム達成率.png`）に。達成率の高い順・棒の中に現在ステージ。MT投影・LINE・Notion貼付用。※Notionの自動計算列(rollup)はAPIから棒グラフ化できないため画像で代替。
- **`build_weekly_deck.py`（週次スライド）** — `members/*.json` 全員から、表紙＋全員ダッシュボード＋各個人スライドの `.pptx` を生成。`pip install python-pptx`。

いずれも同じ `members/*.json` と同じ図解仕様を共有する。

## 使い方

```bash
# 1人分を生成（out/<名前>.svg と .png を出力）
python tools/goalmap/generate_goalmap.py tools/goalmap/members/岡野.json

# 出力先を指定
python tools/goalmap/generate_goalmap.py tools/goalmap/members/岡野.json -o /tmp/okano

# 標準入力から
cat data.json | python tools/goalmap/generate_goalmap.py -
```

- **SVG** は常に出力（ブラウザで開ける／印刷・PDF化も可）。フォントはMac/Win/Linux向けのフォールバック順で埋め込む。
- **PNG** は `cairosvg` があれば出力（`pip install cairosvg`）。日本語ラスタライズには CJK フォントが必要：
  - Ubuntu: `apt-get install -y fonts-noto-cjk`
  - PNG描画時はインストール済みの `Noto Sans CJK JP` を先頭に置いた版で描画する（cairoのCJKフォールバック対策）。

## データモデル（`members/<名前>.json`）

```jsonc
{
  "name": "岡野",
  "note": "wanny",                       // 所属（チーム/グループ。任意）
  "theme": "契約を安定させる（クローザー）",                   // 分野の見出し（任意・下記参照）
  "goal": "客層を問わず自分の型で契約まで運び、成約が安定する",  // 今期の到達点を観察できる状態で書く
  "vision": "半年後にどんな客層でも安定して契約でき、後輩に型を教えられる", // 大枠ゴール（長期・北極星・任意）
  "visionDue": "半年後（2026年内）",       // 大枠の時期ラベル（任意）
  "midGoal": "苦手客層でも型どおり切り返せるようになる", // 中目標（中期のマイルストーン・任意）
  "midDue": "2〜3ヶ月後",                   // 中目標の時期ラベル（任意）
  "why": "…（任意）",
  "currentStage": 2,                       // 1..5（今ここ）
  "cycle": 1,                              // 何周目のゴールか（既定1。2以降はアバターに番号バッジ）
  "logs": [                                // MTログ／議事録（studioで蓄積。任意）
    { "date": "2026-07-01", "text": "決定・FB・メモ", "next": ["次アクション1","次アクション2"] }
  ],
  "startDate": "2026-07-06",               // 計画開始（任意）
  "dueDate": "2026-08-28",                 // ゴール期限（終了日）
  "phases": [                              // 必ず5要素。index0=①(下) … index4=⑤(上)
    { "name": "型を知る", "doneDef": "勝ち負けを言語化できる",
      "due": "2026-07-17",                 // このフェーズの期限（終了日から逆算で初期配分）
      "tasks": [ { "name": "…", "done": true } ] }
  ]
}
```

- **達成率 ＝ `done: true` のタスク数 ÷ 全タスク数 ×100**（自動計算）。
- 時間軸ラベル（クリア/今/＋N週/達成）は `currentStage` から自動。
- `members/_template.json` を雛形にする。

### テーマとゴールの違い

| | テーマ（`theme`） | ゴール（`goal`） |
|---|---|---|
| 役割 | **分野の見出し／分類ラベル** | **今期の到達点（観察できる状態）** |
| 粒度 | ざっくり・継続的（役割の方向性） | 具体的・期限つき（達成したか判定できる） |
| 例 | 「契約を安定させる（クローザー）」 | 「客層を問わず自分の型で契約まで運び、成約が安定する」 |
| 図での場所 | ヘッダーの見出し | 🏆ゴール箱（地図の頂上） |

> **判定の主役はゴール**（達成率もゴール直下のタスクで測る）。テーマは「どの分野の話か」をひと言で表す見出しで、**省略可**。1メンバー＝ゴール1つが基本。テーマとゴールがほぼ同義になるなら、テーマは空でも構わない。

### 目標の階層（大枠 ＞ 中目標 ＞ 今期）— `vision` / `midGoal`

今期ゴール（`goal`）の**さらに上に、大枠ゴールと中目標を積んで“目標の階層”**にする。「今のタスクが、中目標→大枠のどこへ向かう一歩か」を一目で握れる。

- `vision`（string・任意）＝ **大枠ゴール（長期・北極星）**。半年〜1年後の到達像を観察できる状態で。`visionDue`＝時期ラベル（任意）。
- `midGoal`（string・任意）＝ **中目標（中期のマイルストーン）**。今期ゴールと大枠の間の一里塚。`midDue`＝時期ラベル（任意）。
- 図では上から **大枠（紫破線）▸ 中目標（青破線）▸ 今期ゴール（金の実線箱）** を積み、各段を淡い上向き矢印（`その先へ`）で連結。今期ゴールが“通過点”に見える。ヘッダーにも `大枠：…` を1行常時表示。1枚カードにも先頭に最大2行（大枠／中目標）。
- **未設定の段は描画しない**（従来レイアウトのまま・後方互換）。studioエディタの「🌟 大枠ゴール」「🏔 中目標」欄で編集、`generate_goalmap.py` も同仕様でPNGに反映。

### 🔭 俯瞰（フェードアウト）ビュー — studio

studioヘッダーの **`🔭 俯瞰`** を押すと、**タスクを畳んで（淡色にフェードして）、大枠 ▸ 中目標 ▸ 今期ゴールの全体像だけ**を大きな階段で見せる“縮尺切替”ビューに切り替わる。今のタスクが何のための一歩かを、引いて俯瞰できる。

- 下部に5フェーズを淡いチップ（`✓done/total`・現在ステージだけ濃色）で畳んで表示し、「今ここ：③実践」を明示。小アバター＋達成率も添える。
- もう一度 `🔭 俯瞰` で通常地図に戻る（`📱 1枚表示` とは排他）。俯瞰図も `id="gm"` なので **そのまま「PNG出力」で画像化してLINE共有**できる。
- studio専用のビュー切替（`generate_goalmap.py` の静的図は通常の3段レイアウトで出力）。

## 成長キャラ（ゲーム感の演出）

達成度に応じて、ゴール箱の右の相棒が**卵→鳥→不死鳥**に進化する。**1タスク＝餌1つ、1フェーズ食べ切る＝1進化**。studio・PNG・週次スライドすべて同じSVG手描きで出る（絵文字非使用）。

| 形態 | 条件 |
|---|---|
| 🥚 たまご | フェーズ①未完了（餌が入ると殻にヒビ） |
| ひよこ | ①型を知る を完了 |
| 小鶏 | ②練習 を完了 |
| とさか鶏 | ③実践 を完了 |
| 極彩鶏 | ④振り返り を完了（彩り鮮やかな大鶏） |
| 🦚 不死鳥（クジャク風） | 達成率100%（全タスク完了）＝覚醒 |

- 進化の判定は**①から順に「全タスク完了したフェーズ数」**（上表は内部の段階定義。図には形態名は出さない）。脇に **`あと△コで進化`** と `餌 ◯/△・累計 ◯/△` を表示。
- 最終進化（不死鳥）は**クジャク風**：青緑〜金のイリデッセント配色、目玉模様の大扇、特大の翼、冠羽、輝き。

### 周回（何羽目の不死鳥か）— `cycle`

ゴールを達成して**次のゴールを設計すると、アバターは卵に戻る**が、ただのリセットにしない。`cycle`（何周目）を持ち、**2周目以降は卵や鳥の胸（不死鳥は黄金の胸当て）に番号を刻む**（金枠の紋章）。これで「何羽目の不死鳥まで来たか」が一目で分かる（2羽目の不死鳥には「2」）。

- `cycle`（整数・既定`1`）＝今が何周目のゴールか。**1周目は番号なし**（まっさらな初回）、2周目以降は卵〜不死鳥すべての胸に番号を刻む。
- 脇に `○周目・達成 ◯羽`（`達成`＝これまでに孵した不死鳥の数＝`cycle-1`、今回100%ならそれも加算）を表示。
- **studioでの周回運用**：ゴール達成時にエディタの **`🎉 ゴール達成 → 次のゴールへ`** ボタンを押すと、**まずこの周回の地図(PNG)とデータ(JSON)を書き出して**から、`cycle+1`・全タスクのチェック解除・ステージ①へ戻す（ゴール文/テーマ/タスクはそのあと新ゴールに手動で書き換え）。番号は手動でも `周回` 欄で修正可。
- **過去の達成地図を見返す（履歴）**：書き出した2ファイルを **`out/archive/<名前>_cycle<N>.{png,json}`** に保存してコミットすれば、リポジトリ＝履歴台帳になる。CLIなら `python tools/goalmap/archive_cycle.py tools/goalmap/members/<名前>.json` で `out/archive/` に `json/svg/png` を一括出力。PNGはそのまま見返せる。
  - **読み取り専用ビュー**：studioヘッダーの **`🔒 履歴を見る`** で `out/archive/` のJSONを選ぶと、**現在の作業データを上書きせず**にその周回の地図を表示（黄色い「履歴プレビュー」帯＋編集不可）。「× 閉じて現在地に戻る」で元に戻る。ただ見るだけならPNGを開くだけでもよい。
- 全員の今のアバターは studio の**ロスター**で横並び表示（クリックで切替）。
- **アバター単体運用**：studioの「アバターPNG」で、いまのアバターだけのカード（名前＋形態＋達成率＋餌進捗）を書き出せる。「今のフェーズのアバターだけをどこかに見せる」運用向け。
- 実装：`generate_goalmap.py` の `growth()` / `bird_markup()`、studioは同名のJS関数＋`buildAvatarCard()`（仕様一致）。

## 🎙 MTログ / 議事録・録音・次タスク反映（studio）

週次MTの記録と振り返り、そこからの次タスク反映を studio 内で完結させる仕組み。studioの下部「🎙 MTログ / 議事録」パネル（折りたたみ）で操作する。**録音・議事録・タスク化はブラウザ内で動きオフライン対応**。データは `members/<名前>.json` の `logs[]`（テキストのみ）に保存される。

- **録音**：`MediaRecorder`（オフライン）でMT音声を録音→停止で `名前_YYYY-MM-DD_MT.webm`（or `.mp4`）を**ダウンロード**。音声はサイズが大きいのでlocalStorage/リポジトリには保存しない（手元に置き、必要時だけ文字起こし）。※未対応ブラウザでは録音ボタンは出ずメモ入力のみ。
- **議事録（メモ）**：日付＋本文（決定・FB・メモ）＋次アクション（1行1つ）を入力し「この議事録を保存」→ `logs[]` に追加。**過去の議事録は新しい順に一覧**で振り返れる（各項目に削除／MD書き出し）。
- **議事録→次タスク反映（半自動）**：次アクション欄の各行を、選んだフェーズ（既定＝現在ステージ）へ **「＋タスク化」ワンクリック**で追加（AI不要・その場で地図に載る）。過去ログの各次アクションからも個別にタスク化できる。最終確認は人が行う。
- **議事録Markdown書き出し**：`議事録/_template.md` の体裁（決定事項・TODO表・メモ）で `YYYY-MM-DD_<名前>_週次ゴールMT.md` を出力。**`議事録/YYYY/` にコミット**すれば公式議事録として蓄積・全文検索できる。
- **全自動（節目・Claude）**：書き出した議事録Markdown（や録音の文字起こし）をClaudeに渡せば、`members/*.json` のタスク・ステージ・`vision` をまとめて更新→studioで「JSON読込」で反映。※単体HTMLでは高精度な自動文字起こし・AI全自動タスク生成はできないため、そこはClaude/外部ツールに委ねるハイブリッド。
- 履歴（読み取り専用）表示中・サマリー表示中・印刷時はパネルを隠す。

## 図解の確定仕様（厳守）

座標系 viewBox `0 0 680 H`（H可変）。上→下に goal, ⑤, ④, ③, ②, ① の順、矢印は上向き。

| 要素 | 仕様 |
|---|---|
| ヘッダー | 左：`名前（所属）｜テーマ` ／ 右：達成率%＋バー |
| ゴール箱 | 最上部・**輝くグラデーション**（紫#5B4BE0→マゼンタ#B14BE0→金#FFC24B）・金枠#FFD36E・上半分にツヤ。「ゴール」＋ゴール文（白）＋星（SVG手描き） |
| 成長キャラ | ゴール箱の右。達成度で 卵→ひよこ→小鶏→とさか鶏→極彩鶏→不死鳥（上記「成長キャラ」参照）。全媒体SVG手描き |
| フェーズ箱 | x=130, w=210。フェーズ名＋「完了：〜」。左に状態マーカー円。高さ=`max(60, 24+18×タスク数)` |
| 目標時期ピル | 左 x=16, w=80。各フェーズの`due`を「○月第○週」で表示／達成=ゴール。完了済=クリア。**未完了かつ期限超過＝赤＋⚠**、図上部に赤バナー「⚠ 期限に遅れあり」 |
| タスク | 箱の右 x=360〜。チェックボックス＋タスク名。済=塗り＋チェック＋取り消し線 |
| 色 | 済=#1D9E75 / 今=#EF9F27 / これから=#9AA3AD / ゴール=#6357CC |
| フォント | Hiragino Sans → Noto Sans CJK JP → Yu Gothic フォールバック |

## Notion との対応

実データは Notion の 🧭 目標マップ DB / ✅ タスク DB（[`../../ゴールマップ運用.md`](../../ゴールマップ運用.md) §4）。
Notion側のフェーズ「完了の定義」は保存されないため、`doneDef` はこの JSON 側で持つ。

| JSON | Notion 目標マップ |
|---|---|
| `name` | メンバー(select) |
| `theme` | テーマ(title) |
| `goal` | ゴール（観察できる状態）(rich_text) |
| `why` | なぜ今これか(rich_text) |
| `currentStage` | 現在ステージ(select ①..⑤) |
| `phases[].tasks[]` | ✅ タスク DB（ステージ select、ステータス status＝完了で `done:true`） |
