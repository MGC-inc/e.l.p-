# イノベラ.md — 電話録音の要約・通知フロー

> イノベラ（INNOVERA: クラウドPBX/通話録音）の録音を「どう要約して」「どこに・誰に送るか」を定義するファイル。

## 1. フロー全体像

```
電話着信/発信（イノベラで録音）
  → 録音データ取得
  → 文字起こし
  → 下記テンプレで要約
  → LINEで東さんへ通知
  → 通話ログ/YYYY/YYYY-MM-DD_相手.md として保存
  → 重要案件は 議事録/ に転記
```

## 2. 要約テンプレート

```
【通話要約】YYYY-MM-DD HH:MM
■ 相手: （会社名・氏名）
■ 用件: （1行）
■ 内容:
- 要点1
- 要点2
■ 対応が必要なこと:
- [ ] （誰が・いつまでに・何を）
■ 緊急度: 高 / 中 / 低
```

## 4. 技術メモ

- イノベラのAPI仕様・認証情報の保管場所は [API一覧.md](./API一覧.md) を参照（キー実体はローカル `.env`。リポジトリに置かない）
- 自動化パイプライン（録音取得→文字起こし→構造化→DB反映）の実装状況: **実装済み（2026-06-10）**
  - スクリプト: `scripts/innovera_pipeline.py`（CDR検索→録音DL→Gemini 2.5-flash文字起こし+19項目抽出→Supabase call_logs へupsert）
  - プロンプト: `scripts/gemini_prompt.txt`（車トラブル受付の19項目TSV仕様）
  - 対象回線: circuit 1=車トラブル相談窓口(05088960087) / 2=車トラブル受付センター(05088906176)
  - アプリ表示: 業務ポータル `/calls`（文字起こし全文・要約・19項目・スプレッドシート用TSVコピー）
  - 未連携: LINE通知（東さん）はこのパイプラインには未接続。要約のDB反映までが現状
  - 文字起こしエンジン: Gemini（旧方針のmlx-whisperから変更）

### 自動化パイプラインの実装方針（仕様書ベース）

```
1. 電話履歴検索API（?ckey=cdr&akey=search）で終話済み・録音あり（record_file_flg=1）の履歴を取得
2. 録音ファイルURL検索API（?ckey=cdr&akey=record）で録音URLを取得 ※終話後のみ（通話中アクセスでファイル破損）
3. 録音を文字起こし → セクション2のテンプレで要約
4. LINEで東さんへ通知 ＋ 通話ログ/ に保存
```

- 代替案: イノベラ標準の「着信連携（webAPI実行版）」で応答終了時/不在終了時に当社エンドポイントへ通話情報（uniqid等）をPOSTさせる構成も可能（URL設定はプロディライト側作業）
- レート制限: 5分200リクエスト/IP に注意
