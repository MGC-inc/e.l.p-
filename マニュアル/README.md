# マニュアル/ — 業務マニュアル

> 清掃業務の標準手順・品質基準を集約するフォルダ。
> 目的: 新人オンボーディングの短縮と、業務の属人化解消。

## 1. ルール

- **1業務1ファイル**: `マニュアル/業務名.md`（例: `定期清掃_オフィス.md`、`見積もり作成.md`）
- **テンプレート**: [_template.md](./_template.md) をコピーして使う
- 手順が変わったら即更新し、`docs(マニュアル): 〇〇の手順を変更` でコミット
- 「誰がやっても同じ品質になる」粒度で書く — 暗黙知（コツ・注意点）こそ言語化する

## 3. 何からマニュアル化するか（優先順位の付け方）

1. **頻度が高い業務** — 毎日・毎週やること
2. **特定の人しかできない業務** — 属人化リスクが高いもの
3. **ミスが起きたことのある業務** — 再発防止
4. **新人が最初に覚える業務** — オンボーディング用
