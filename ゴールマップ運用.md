# ゴールマップ運用.md — 週次MTG向け 個人ゴール図解の運用

> 営業マネージャーが、各メンバーの目標を「**下から上へ登る地図**」の図解にして、
> **毎週・各個人に進捗を見せて握る**ための運用ルール。
> 図解の確定仕様・設計思想は [`tools/goalmap/README.md`](./tools/goalmap/README.md) と
> 各メンバーの `tools/goalmap/members/*.json` を正とする。

## 1. このループの全体像

```
 Notion（営業実績の実データ：目標マップ / タスク）
        │  ① 進捗を吸い上げ
        ▼
 ユーザー（マネージャー）─── ② このリポジトリに「個人の進捗」を共有
        │                     軌道修正・ゴール設計の変更があるときだけフィードバック
        ▼
 Claude Code ─── ③ tools/goalmap/members/<名前>.json を更新
        │        ④ generate_goalmap.py で図解（SVG/PNG）を生成
        ▼
 週次MTG ─── ⑤ 各メンバーに図解を提示して、現在地とゴールを握り直す
```

- **データの正は Notion**（🧭 目標マップ DB / ✅ タスク DB）。
- **図解の元データの正はこのリポジトリの `members/<名前>.json`**。Notionから写し、MTでの合意・軌道修正を反映する手元の決定版。
- **マネージャーがすること**: 進捗をここに共有し、**軌道修正やゴール設計を変える時だけフィードバック**する。
- **Claudeがすること**: 進捗を `members/*.json` に反映し、図解を出力する。

## 2. 二つの使い方

### A. MT中にその場で編集する（推奨・主役） — `tools/goalmap/goalmap-studio.html`

理想の運用は「**先週の図解を見ながら、できた／できていないを確認し、できたタスクにチェックを入れる。MTが終わった時には翌週の図解が完成している**」。これを叶えるのがブラウザ版スタジオ。

1. `tools/goalmap/goalmap-studio.html` をブラウザで開く（インストール不要）
2. メンバーを選ぶ（先週の状態は自動保存＝localStorageから復元。無ければ「JSON読込」で `members/<名前>.json` を読む）
3. **MT中に左でチェック／編集** → 右の図解と達成率がリアルタイム更新
   - できたタスクにチェック、ステージが進んだら「現在ステージ」を変更
   - 軌道修正・ゴール変更はその場でゴール文やタスクを編集
4. 終了時：「PNG出力」で記録、「JSON保存」で `members/<名前>.json` を更新（リポジトリにコミットして履歴に残す）

### B. まとめてバッチ生成する — `generate_goalmap.py`

チャットで進捗を共有してもらい、Claudeが反映して出力する従来ループ。記録・一括出力向き。

1. 対象メンバーの最新状況を受け取る（Notion or チャット）
2. `tools/goalmap/members/<名前>.json` を更新（`done` / `currentStage` / `goal` / `doneDef`）
3. 生成: `python tools/goalmap/generate_goalmap.py tools/goalmap/members/<名前>.json`
4. `out/<名前>.svg` / `.png` を提示し、達成率・今ここ・ネクストを握る

### C. 週次スライドに全員分を格納する — `build_weekly_deck.py`

週次MTGのスライドに各個人の図解を載せる用。`members/*.json` 全員から、
**表紙＋全員ダッシュボード（達成率・現在ステージ一覧）＋各メンバー1枚（図解＋フォーカス/ネクスト）**
の `.pptx` を生成する。

```bash
python tools/goalmap/build_weekly_deck.py            # 全員 → out/週次ゴール進捗_YYYY-Www.pptx
python tools/goalmap/build_weekly_deck.py 岡野 宮腰   # 指定メンバーのみ
```

> A・B・Cは同じ `members/*.json` と同じ図解仕様を共有する。MT中の編集はA、記録はB、配布スライドはC。

## 3. 達成率と図解の不変ルール（変更しない）

- **ゴールから逆算**：図は下から上へ積み上げ、最上部がゴール（読みは上→下、進捗は下→上）
- **観察できる完了定義**：ゴールもフェーズの完了も、数値でなく観察できる状態で書く
- **5フェーズの背骨**：①型を知る → ②練習 → ③実践 → ④振り返り → ⑤自走
- **達成率 ＝ 完了タスク数 ÷ 全タスク数 ×100**（`done: true` の割合）
- **色＝状態**：済=ティール+チェック / 今=アンバー(2px) / これから=グレー / ゴール=パープル
- **目標時期（固定期限）**：各フェーズに`due`（期限）を持ち「○月第○週」で表示。ゴール終了日(`dueDate`)から逆算して初期配分。**未完了かつ期限超過＝赤＋⚠、図上部に「⚠ 期限に遅れあり」バナー**で強調告知。
- **延期運用**：期限に間に合わなければ週次MTで延期。studioでフェーズの日付を変更、または「＋1週」ボタンで**そのフェーズ以降の期限をまとめて1週うしろへ**ずらす。
- **周回（卵に戻ってもリセットにしない）**：ゴール達成→次のゴール設計でアバターは卵に戻るが、`cycle`（何周目）で**2周目以降は卵や鳥の胸（不死鳥は黄金の胸当て）に番号が刻まれ**「何羽目の不死鳥まで来たか」が一目で分かる。studioの **`🎉 ゴール達成 → 次のゴールへ`** ボタンで `cycle+1`・チェック解除・①へ戻す（ゴール/タスクは新ゴールに書き換え）。
- **過去の達成地図を見返す（履歴）**：「次のゴールへ」を押すと、**リセット前にその周回の地図(PNG)とデータ(JSON)が書き出される**。`tools/goalmap/out/archive/<名前>_cycle<N>.{png,json}` に保存してコミットすれば、リポジトリが履歴台帳になり過去の達成地図をいつでも見返せる（CLIは `python tools/goalmap/archive_cycle.py members/<名前>.json`）。Notionは「完了タスクの行」が残る補助的ログ。

### Notion運用スキーム（タスクは担当のフェーズ・役職で都度見直す）

**原則：1タスク = 1メンバー（「目標」リレーション必須）。** 役割・フェーズでタスク内容が違うのは当然なので、共有せず各人が自分のタスク行を持つ（重複OK）。全タスクに「目標」を付ければ、目標マップの **`進捗率`rollup が自動計算**され、図解の達成率と一致する。※2026-06-24に全タスクの目標紐付けを整備済み（未紐付け0件）。

**目標マップDBの構造（既存・自動集計あり）**
`メンバー` / `役割`(クローザー・管理・育成担当・新人) / `現在ステージ`(①〜⑤) / `今週のフォーカス` / `ネクストアクション` / `改訂日` / rollup `進捗率`・`完了タスク数`・`総タスク数`。

**GMタスクDBの運用ビュー（作成済み）**
| ビュー | 設定 | 用途 |
|---|---|---|
| メンバー別ボード | グループ=目標 | 各人のタスクをカラム表示。見直しの主戦場 |
| フェーズ別ボード | グループ=ステージ | ①〜⑤のカンバン |
| 今週フォーカス | ソート=ステージ／列=タスク名・ステージ・ステータス・目標・担当 | 今週やること |

> ⚠️ `今週フォーカス` の「ステータスが完了でない」フィルタはAPI（MCP）で設定不可のため、**Notion UIで1クリック追加**が必要（フィルタ→ステータス→完了を除外）。

**見直しサイクル（“都度見直す”の中身）**
週次/隔週で各メンバーのボードを開き → ①フェーズが進んだら `現在ステージ` 更新＋次フェーズのタスク追加・不要削除 → ②`今週のフォーカス`・`ネクストアクション` 書き換え → ③`改訂日` 更新。新メンバーは役割に応じたタスクセットを複製して紐付け（役割別テンプレはNotion UIのDBテンプレートで用意）。

### 図解をNotionのゴールページに貼る（ネイティブ図解）

図解PNGはGitHub（プライベート）にあるためNotionへのURL埋め込み・画像アップロードはできない。代わりに **Notionネイティブの図解**（callout＋Mermaidフローチャート＋ステージ別チェックリスト）を各メンバーのゴールページ本文に挿入する。

```bash
python tools/goalmap/notion_block.py tools/goalmap/members/岡野.json   # 貼り付け用Markdownを出力
```

出力をNotionゴールページ本文に挿入（Claude経由なら `notion-update-page` の insert_content）。Mermaidブロックは Notion上で図としてレンダリングされる。

- **更新時**：ページ本文を入れ替える（古い図解ブロックを消して再挿入 or `replace_content`）。再挿入だけだと重複するので注意。
- **正確なスタイルのPNGも置きたい場合**：`out/<名前>.png` を手動でドラッグ＆ドロップ（1人1回）。
- 投入済み（2026-06-25）：岡野・宮腰のゴールページに挿入済み。
