# ゴールマップ運用.md — 週次MTG向け 個人ゴール図解の運用

> 営業マネージャーが、各メンバーの目標を「**下から上へ登る地図**」の図解にして、
> **毎週・各個人に進捗を見せて握る**ための運用ルール。
> 図解の確定仕様・設計思想は [`tools/goalmap/README.md`](./tools/goalmap/README.md) と
> 各メンバーの `tools/goalmap/members/*.json` を正とする。

## 1. 運用方針（確定）— studio中心・週1編集・画像で共有

```
 goalmap-studio.html（手元のブラウザ｜編集の本拠地）
        │  週1：MTで✓・調整 → 画像出力
        ▼
 画像（PNG）を各担当へLINEで共有
        │
        ▼
 Notion（🧭 目標マップ / ✅ タスク）＝ データ蓄積・参照用（スマホ閲覧）
```

- **編集はstudioだけ・週1回**。MT中に✓を入れ・調整し、終わったら**画像（PNG）を出して各担当にLINEで共有**する。これがメイン運用。
- **データの正＝studio**（手元の `members/<名前>.json`／ブラウザの自動保存）。Notionは**蓄積・参照用**（主にスマホで見る台帳）で、毎週は触らない。
- **Claudeは“メイン”ではない**：毎週は不要。**まとめて変更・Notionへ反映・履歴コミット・ツール改修・議事録反映**など、たまの重い作業の時だけ使う。
- studio・Notionは別々のデータ置き場。**毎週の✓はstudioだけ**で行う（両方バラバラに触らない）。Notionを最新化したい時は節目でまとめて反映（Claudeに「Notionに反映して」）。

## 2. 毎週の手順（週1・約10分｜studioが主役）

`tools/goalmap/goalmap-studio.html` をブラウザで開く（インストール不要・前回の状態は自動復元）。

**各メンバーごとに（MT中）**
1. ドロップダウン／ロスターで**メンバーを選ぶ**
2. 先週からできた**タスクに✓**（右の図解と達成率が即更新）
3. **来週のタスク・方向性をすり合わせ** → その場で編集（追加／書換／×削除）
4. ステージが進んだら**現在ステージ**を変更
5. 遅れていたら**期限を延期**（日付変更 or「＋1週」。超過は赤で警告）
6. 必要なら**大枠ゴール（🌟北極星）・中目標（🏔）を確認・調整**（今のタスクが中目標→大枠のどこへの一歩か握る。**`🔭 俯瞰`** でタスクを畳み全体像だけを見せると握りやすい）
7. ゴール達成なら **🎉 次のゴールへ**（卵に戻り周回番号UP・地図は自動アーカイブ）

**記録（🎙 MTログ / 議事録・任意）**
8. 下部「🎙 MTログ / 議事録」を開き、必要なら **● 録音**（停止で音声ファイルDL＝後で文字起こし可）
9. **議事録メモ**（決定・FB・次アクション）を入力 →「この議事録を保存」（過去分は新しい順に振り返れる）
10. 次アクションは **「＋タスク化」** で選んだフェーズのタスクに即反映（AI不要）。公式議事録に残すなら「議事録をMarkdownで書き出し」→ **`議事録/2026/` にコミット**（Claudeに「コミットして」）

**共有（各担当へ）**
11. **「📱 1枚表示」→「PNG出力」**（または画面スクショ）で1枚画像を出す → **LINEで送る**
   - フルの“登る地図”を見せたい時は通常表示のまま「PNG出力」
   - 文字をクッキリ見せたい時はLINEで**「ファイル」送信**（原画質）

**終了時**
12. studioは自動保存（localStorage）。**毎回のコミットは不要**。正データのバックアップ／履歴に残したい時だけ「JSON保存」→ Claudeに「コミットして」。

### （任意）たまにやる
- **Notionを最新化**：studioの内容をNotionへ反映 → Claudeに「Notionに反映して」
- **全員分を一括画像化**：`summary_cards.py`（1枚カード全員分）／`generate_goalmap.py`（フル図解）／`team_chart.py`（達成率の棒グラフ・任意）
- ゴール/タスクの大きな組み替え・ツールの見た目変更・議事録からの反映 → Claude

### （任意）バッチ生成・配布ツール

毎週はstudioだけでよいが、まとめて画像化したい時の補助（Claudeに頼んでもよい）。すべて同じ `members/*.json` と同じ図解仕様を共有する。

- `python tools/goalmap/generate_goalmap.py members/<名前>.json` … フル図解（`out/<名前>.svg/.png`）。`--summary` でスマホ1枚カード。
- `python tools/goalmap/summary_cards.py` … 全員の1枚カードを `out/summary/` に一括出力。
- `python tools/goalmap/team_chart.py` … チーム達成率の棒グラフ（任意）。
- `python tools/goalmap/build_weekly_deck.py` … 全員を1冊の `.pptx`（投影・配布が必要な時のみ）。

## 3. 達成率と図解の不変ルール（変更しない）

- **ゴールから逆算**：図は下から上へ積み上げ、最上部がゴール（読みは上→下、進捗は下→上）
- **目標の階層（大枠＞中目標＞今期）**：今期ゴールの上に、**大枠ゴール（長期・北極星＝`vision`）と中目標（中期のマイルストーン＝`midGoal`）**を積み、`その先へ` の淡い矢印で連結。今のタスクが中目標→大枠のどこへ向かう一歩かを一目で握る。未設定の段は描画しない（後方互換）。studioの **`🔭 俯瞰`** ボタンで、タスクを畳んで目標の全体像（大枠▸中目標▸今期）だけを俯瞰でき、そのままPNG出力で共有できる。
- **観察できる完了定義**：ゴールもフェーズの完了も、数値でなく観察できる状態で書く
- **5フェーズの背骨**：①型を知る → ②練習 → ③実践 → ④振り返り → ⑤自走
- **達成率 ＝ 完了タスク数 ÷ 全タスク数 ×100**（`done: true` の割合）
- **色＝状態**：済=ティール+チェック / 今=アンバー(2px) / これから=グレー / ゴール=パープル
- **目標時期（固定期限）**：各フェーズに`due`（期限）を持ち「○月第○週」で表示。ゴール終了日(`dueDate`)から逆算して初期配分。**未完了かつ期限超過＝赤＋⚠、図上部に「⚠ 期限に遅れあり」バナー**で強調告知。
- **延期運用**：期限に間に合わなければ週次MTで延期。studioでフェーズの日付を変更、または「＋1週」ボタンで**そのフェーズ以降の期限をまとめて1週うしろへ**ずらす。
- **周回（卵に戻ってもリセットにしない）**：ゴール達成→次のゴール設計でアバターは卵に戻るが、`cycle`（何周目）で**2周目以降は卵や鳥の胸（不死鳥は黄金の胸当て）に番号が刻まれ**「何羽目の不死鳥まで来たか」が一目で分かる。studioの **`🎉 ゴール達成 → 次のゴールへ`** ボタンで `cycle+1`・チェック解除・①へ戻す（ゴール/タスクは新ゴールに書き換え）。
- **過去の達成地図を見返す（履歴）**：「次のゴールへ」を押すと、**リセット前にその周回の地図(PNG)とデータ(JSON)が書き出される**。`tools/goalmap/out/archive/<名前>_cycle<N>.{png,json}` に保存してコミットすれば、リポジトリが履歴台帳になり過去の達成地図をいつでも見返せる（CLIは `python tools/goalmap/archive_cycle.py members/<名前>.json`）。studioの **`🔒 履歴を見る`** で、現在の作業データを上書きせず読み取り専用で過去周回を開ける。Notionは「完了タスクの行」が残る補助的ログ。
- **MTログ / 議事録・録音（studioに内蔵）**：週次MTの録音（オフライン→音声DL）・議事録メモ（`logs[]` に蓄積・新しい順に振り返り）・**次アクション→フェーズのタスク化**（ワンクリック）を studio 下部パネルで完結。議事録は `議事録/_template.md` 体裁で **Markdown書き出し→`議事録/YYYY/` にコミット**すれば公式議事録として蓄積・検索できる。録音の文字起こしや議事録からのタスク一括反映など重い処理は Claude に渡す（単体HTMLでは自動文字起こし・AI全自動反映は不可）。詳細は [`tools/goalmap/README.md`](./tools/goalmap/README.md)。

### Notion運用スキーム（タスクは担当のフェーズ・役職で都度見直す）

**原則：1タスク = 1メンバー（「目標」リレーション必須）。** 役割・フェーズでタスク内容が違うのは当然なので、共有せず各人が自分のタスク行を持つ（重複OK）。全タスクに「目標」を付ければ、目標マップの **`進捗率`rollup が自動計算**され、図解の達成率と一致する。※2026-06-24に全タスクの目標紐付けを整備済み（未紐付け0件）。

**目標マップDBの構造（既存・自動集計あり）**
`メンバー` / `役割`(クローザー・管理・育成担当・新人) / `現在ステージ`(①〜⑤) / `今週のフォーカス` / `ネクストアクション` / `改訂日` / rollup `進捗率`・`完了タスク数`・`総タスク数`。

**GMタスクDBの運用ビュー（作成済み）**
| ビュー | 設定 | 用途 |
|---|---|---|
| メンバー別ボード | グループ=目標 | 各人のタスクをカラム表示。見直しの主戦場 |
| フェーズ別ボード | グループ=ステージ | ①〜⑤のカンバン |
| 今週フォーカス | ソート=ステージ／列=タスク名・ステージ・ステータス・目標・担当 | 今週やること |

> ⚠️ `今週フォーカス` の「ステータスが完了でない」フィルタはAPI（MCP）で設定不可のため、**Notion UIで1クリック追加**が必要（フィルタ→ステータス→完了を除外）。

**見直しサイクル（“都度見直す”の中身）**
週次/隔週で各メンバーのボードを開き → ①フェーズが進んだら `現在ステージ` 更新＋次フェーズのタスク追加・不要削除 → ②`今週のフォーカス`・`ネクストアクション` 書き換え → ③`改訂日` 更新。新メンバーは役割に応じたタスクセットを複製して紐付け（役割別テンプレはNotion UIのDBテンプレートで用意）。

### 図解をNotionのゴールページに貼る（ネイティブ図解）

図解PNGはGitHub（プライベート）にあるためNotionへのURL埋め込み・画像アップロードはできない。代わりに **Notionネイティブの図解**（callout＋Mermaidフローチャート＋ステージ別チェックリスト）を各メンバーのゴールページ本文に挿入する。

```bash
python tools/goalmap/notion_block.py tools/goalmap/members/岡野.json   # 貼り付け用Markdownを出力
```

出力をNotionゴールページ本文に挿入（Claude経由なら `notion-update-page` の insert_content）。Mermaidブロックは Notion上で図としてレンダリングされる。

- **更新時**：ページ本文を入れ替える（古い図解ブロックを消して再挿入 or `replace_content`）。再挿入だけだと重複するので注意。
- **正確なスタイルのPNGも置きたい場合**：`out/<名前>.png` を手動でドラッグ＆ドロップ（1人1回）。
- 投入済み（2026-06-25）：岡野・宮腰のゴールページに挿入済み。
