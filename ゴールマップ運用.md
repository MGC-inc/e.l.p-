# ゴールマップ運用.md — 週次MTG向け 個人ゴール図解の運用

> 営業マネージャーが、各メンバーの目標を「**下から上へ登る地図**」の図解にして、
> **毎週・各個人に進捗を見せて握る**ための運用ルール。
> 図解の確定仕様・設計思想は [`tools/goalmap/README.md`](./tools/goalmap/README.md) と
> 各メンバーの `tools/goalmap/members/*.json` を正とする。

## 1. このループの全体像

```
 Notion（営業実績の実データ：目標マップ / タスク）
        │  ① 進捗を吸い上げ
        ▼
 ユーザー（マネージャー）─── ② このリポジトリに「個人の進捗」を共有
        │                     軌道修正・ゴール設計の変更があるときだけフィードバック
        ▼
 Claude Code ─── ③ tools/goalmap/members/<名前>.json を更新
        │        ④ generate_goalmap.py で図解（SVG/PNG）を生成
        ▼
 週次MTG ─── ⑤ 各メンバーに図解を提示して、現在地とゴールを握り直す
```

- **データの正は Notion**（🧭 目標マップ DB / ✅ タスク DB）。
- **図解の元データの正はこのリポジトリの `members/<名前>.json`**。Notionから写し、MTでの合意・軌道修正を反映する手元の決定版。
- **マネージャーがすること**: 進捗をここに共有し、**軌道修正やゴール設計を変える時だけフィードバック**する。
- **Claudeがすること**: 進捗を `members/*.json` に反映し、図解を出力する。

## 2. 二つの使い方

### A. MT中にその場で編集する（推奨・主役） — `tools/goalmap/goalmap-studio.html`

理想の運用は「**先週の図解を見ながら、できた／できていないを確認し、できたタスクにチェックを入れる。MTが終わった時には翌週の図解が完成している**」。これを叶えるのがブラウザ版スタジオ。

1. `tools/goalmap/goalmap-studio.html` をブラウザで開く（インストール不要）
2. メンバーを選ぶ（先週の状態は自動保存＝localStorageから復元。無ければ「JSON読込」で `members/<名前>.json` を読む）
3. **MT中に左でチェック／編集** → 右の図解と達成率がリアルタイム更新
   - できたタスクにチェック、ステージが進んだら「現在ステージ」を変更
   - 軌道修正・ゴール変更はその場でゴール文やタスクを編集
4. 終了時：「PNG出力」で記録、「JSON保存」で `members/<名前>.json` を更新（リポジトリにコミットして履歴に残す）

### B. まとめてバッチ生成する — `generate_goalmap.py`

チャットで進捗を共有してもらい、Claudeが反映して出力する従来ループ。記録・一括出力向き。

1. 対象メンバーの最新状況を受け取る（Notion or チャット）
2. `tools/goalmap/members/<名前>.json` を更新（`done` / `currentStage` / `goal` / `doneDef`）
3. 生成: `python tools/goalmap/generate_goalmap.py tools/goalmap/members/<名前>.json`
4. `out/<名前>.svg` / `.png` を提示し、達成率・今ここ・ネクストを握る

> A・Bは同じ `members/*.json` と同じ図解仕様を共有する。MTはA、自動化・記録はBが向く。

## 3. 達成率と図解の不変ルール（変更しない）

- **ゴールから逆算**：図は下から上へ積み上げ、最上部がゴール（読みは上→下、進捗は下→上）
- **観察できる完了定義**：ゴールもフェーズの完了も、数値でなく観察できる状態で書く
- **5フェーズの背骨**：①型を知る → ②練習 → ③実践 → ④振り返り → ⑤自走
- **達成率 ＝ 完了タスク数 ÷ 全タスク数 ×100**（`done: true` の割合）
- **色＝状態**：済=ティール+チェック / 今=アンバー(2px) / これから=グレー / ゴール=パープル
- **時間軸**：左ピルに クリア / 今 / ＋N週 / 達成

### ⚠️ 既知のデータ課題（達成率の精度に直結）

タスクDBの「目標」リレーションが**ほぼ全件 未設定（null）**。このままだと
「目標＝対象ゴールで絞ってタスク側で達成率を集計」が機能しない。当面は
`members/*.json` を手元の正として運用し、Notion側は以下のいずれかで整備する：

1. 各タスクの「目標」リレーションを対象ゴールに紐付ける（推奨。タスク側集計が復活）、または
2. タスクDBに「メンバー」selectを追加し、メンバー軸で集計する

> どちらに進めるかはマネージャー判断（Notion実データの更新を伴うため、勝手に変更しない）。
